搭建hexo个人博客遇到的一些问题

发表于 2020-03-26
在使用github+hexo next主题搭建个人博客时遇到的一些问题以及解决方案（保持更新）

安装hexo即执行npm install时进度条停止

部分的安装无响应以及下载缓慢是因为网速问题导致的，使用淘宝镜像cnpm可以解决这一问题，缺点是淘宝镜像的部分依赖包不完整。

使用淘宝镜像仍然发现不能正常安装，经过尝试发现是hexo路径的问题。最初，我将hexo文件夹建立在C:\Program Files的目录之下，致使安装不能成功。将其安装在用户文件夹之下时，问题就解决了。

hexo new ‘文章’ 没有生成对应的.md文件

绝大多数的hexo命令需要先进入到…\hexo（自命名）\blog的文件夹中，而不是在hexo文件夹下直接操作。以及站点根目录同样指的是blog而不是hexo，新手容易混淆这一点。

报错信息：

C:\Users\administrator\hexo>hexo new '新文章'
Usage: hexo <command>

Commands:
help Get help on a command.
init Create a new Hexo folder.
version Display version information.

Global Options:
--config Specify config file instead of using _config.yml
--cwd Specify the CWD
--debug Display all verbose messages in the terminal
--draft Display draft posts
--safe Disable all plugins and scripts
--silent Hide output on console

For more help, you can use 'hexo help [command]' for the detailed information
or you can check the docs: http://hexo.io/docs/

输入命令 cd blog 操作进入到子文件夹就可以正常使用hexo new了

发布新博客后主页空白

当我发布上一篇博客后，发现主页内容全部空白，以为是误删了_config.yml中的配置。尝试删除了这篇博客后，主页又可以正常显示了，那么问题就锁定在这篇博客。

在网上搜索了一些解决办法，关于此类问题还是比较少的。其中的一个说法是hexo冒号后是必须要有空格的，我仔细检查后，发现问题不在这里。也可以采用将博客内容复制到能够正常运行的博客下面，测试是否能够正常运行的办法，如果不能，则说明是内容中有违法的输入。

我将原博客中可能有问题的地方逐个粘贴的测试文档中，排查问题后终于找到了问题所在：hexo的md普通文本中是不能有如下这样的符号的，具体原因可能与html有关，尚不明晰。

为博客添加评论区

disqus比较适用于英文博客，而适用于中文博客的多说与网易云跟帖相继关闭，因此选择的来必力，评论管理功能比较便捷，next主题中也已经有相关的配置。缺点是来必力加载较慢，更好的评论系统还在进一步探索中。

hexo插入本地图片

插件安装：

把主页配置文件_config.yml 里的post_asset_folder: 这个选项设置为true

安装插件，在blog中执行如下命令

npm install hexo-asset-image --save

安装完成后，再hexo new 新文章 时就会生成一个同名的文件夹

方法一：

将图片放在该文件夹中，用markdown的格式插入图片，hexo插入图片使用的是相对路径，所以直接用图片名即可，无需标注前面的/source/posts等。

![](/01.png)

注：我使用的markdown编辑器是Typora，它会自动提醒你选择路径，通过该方法选择之后代码将修改为完整的路径，这样的图片在博客预览中看不出问题但是在hexo中是不显示的。反之，以正确的格式手动输入图片路径，在Typora中是无法正常预览的，会出现下图的情况。所以，手动输入相对路径，比较安全。

方法二：

在source下创建images文件夹，将用到的图片保存在这个文件夹中，与直接在同名文件夹使用区别是需要标注/images，因为hexo中引用图片使用的是相对路径，images文件夹服务的是整个站点而不仅仅是这一篇博客。
